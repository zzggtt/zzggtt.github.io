从简单的两行代码学起

Author: ZGGTT Date: 2023-01-10 Reading Time:10.8 mins words:2819 Views: -

首先我们从下面一段简单的代码出发，里面涉及到了多个知识点：

data1 = 3.14
data2 = data1

赋值语句的执行顺序是什么？

class Gizmo:
    def __init__(self) -> None:
        print('Gizmo id: {}'.format(id(self)))  # 实例化对象时,打印对象的标识(内存地址)


x = Gizmo() # Gizmo id: 140650247909160
# 先打印：Gizmo id: 140650248064696
# 然后抛出异常:TypeError: unsupported operand type(s) for *: 'Gizmo' and 'int'
y = Gizmo() * 10

从上面的例子中我们可以看出，为了理解Python中赋值语句，应该始终先读右边。对象在右边先创建或获取，在此之后左边的变量才会绑定到对象上。

对象底层实现原理

PyObject

先来看看Python3.7的源码，可以参考下面链接：https://github.com/python/cpython/blob/949fe976d5c62ae63ed505ecf729f815d0baccfc/Include/object.h

可以看到PyObjcet和PyVarObject中有四部分内容：

  • **_PyObject_HEAD_EXTRA：**宏定义，包含上一个、下一个对象指针，用于构造双向循环链表
  • **Py_ssize_t ob_refcnt：**对象引用计数器，用于垃圾回收
  • **_typeobject *ob_type：**对象类型
  • ob_size：容器中对象个数

PyFloatObject

再看下Python内置的数据类型float的底层实现原理，可参考下面链接：https://github.com/python/cpython/blob/949fe976d5c62ae63ed505ecf729f815d0baccfc/Include/floatobject.h

  • **PyObject_HEAD：**是在object.h中的宏定义，本质是PyObject
  • **ob_fval：**用于保存float类型对象的值

PyListObject

再看下Python内置的数据类list的底层实现原理，可参考下面链接：
https://github.com/python/cpython/blob/949fe976d5c62ae63ed505ecf729f815d0baccfc/Include/listobject.h

  • **PyObject_VAR_HEAD：**是在object.h中的宏定义，本质是PyVarObject
  • **PyObject **ob_item：**指针列表，里面的每一个指针都指向列表对象的标识（内存地址）
  • **Py_ssize_t allocated：**存储了列表已分配的内存空间大小

总结

每个Python对象都有标识（内存地址）、类型、值、双向循环链表上下指针、引用计数器（后两个用于Python内部实现垃圾回收）。

变量是什么？

人们经常使用“变量是盒子”这样的比喻，但是这有碍于理解面向对象语言中的引用式变量。Python变量类似于Java中的引用变量，因此最好把它们理解为附加在对象上的标注（类似于便利贴）。
例如下面的例子，就无法使用“变量是盒子”做解释：

a = [1, 2, 3]
b = a
a.append(4)
print(b)  // [1, 2, 3, 4]

在==和is之间选择

:::info
==比较的是对象的值（对象中保存的数据），而is比较的是对象的标识（内存地址）
:::

l1 = [1, 2, "hello"]
l2 = [1, 2, "hello"]

l1 == l2  // True
l1 is l2  // False

is 运算符比 ==速度快，因为它不能重载，所以Python不用寻找并调用特殊方法，而是直接比较对象的标识（内存地址）。而a==b是语法糖，等同于 a.eq(b)，可能会涉及大量处理工作，例如：比较大型集合或嵌套层级深的结构时。

深浅拷贝

深浅拷贝的目标是可变类型对象，对不可类型进行深浅拷贝没有意义。

浅拷贝

如果我们需要复制一个列表对象，最简答的方法就是通过内置的类型构造方法，例如：

l1 = [3, [55, 66], (7, 8, 9)]
l2 = list(l1)  # list(l1) -> 创建l1的副本

print(l1 == l2)  # True  -> 副本与原列表的值相同
print(l1 is l2)  # Flase -> 但是l1和l2是不同的对象
print(l1[0] is l2[0])  # True  -> 副本中的容器还是原容器的引用

浅拷贝：即复制了最外层的容器，副本中的元素还是原容器的引用。如果容器中所有元素都是不可变类型，那么这样没有问题，如果是可变类型，可能得到的结果不是我们想要的，例如：

l1 = [3, [55, 66], (7, 8, 9)]
l2 = list(l1)  # list(l1) -> 创建l1的副本

l1.append(100)  # 把100追加到原容器中，不会影响l2
print(l1)  # [3, [55, 66], (7, 8, 9), 100]
print(l2)  # [3, [55, 66], (7, 8, 9)]

l1[1].remove(55)  # 把原容器中l[1]的55移除，会对l2产生影响，因为l2[1]的列表对象引用是原容器的列表对象引用
print(l1)  # [3, [66], (7, 8, 9), 100]
print(l2)  # [3, [66], (7, 8, 9)]

l2[1] += [33, 22]  # 对可变类型来说 += 是修改列表
l2[2] += (10, 11)  # 对不可变类型来说 += 相当于创建一个新的对象，再把l2[2]指向该对象
print(l1)  # [3, [66, 33, 22], (7, 8, 9), 100]
print(l2)  # [3, [66, 33, 22], (7, 8, 9, 10, 11)]

深拷贝

copy模块提供的deepcopy和copy函数能够为任意对象做深浅复制：

import copy

l1 = ['alice', 'bill', 'david', ['hello', 'world']]
l2 = copy.copy(l1)
l3 = copy.deepcopy(l1)
print(l1 is l3)  # Flase
print(l1[0] is l3[0])  # True  -> 深复制操作：原容器中元素是不可变类型，还是原来的引用不影响
print(l1[3] is l3[3])  # False  -> 深复制操作：如果原容器中元素是可变类型也会拷贝一份，而不是引用

Python的垃圾回收机制

程序运行过程中会申请大量的内存空间，而对于一些无用的内存空间如果不及时清理的话会导致内存使用殆尽（内存溢出），导致程序崩溃，因此管理内存是一件重要且繁杂的事情。通常来说，对于Python这种高级别的语言，开发者完成可以不用关心其内部的垃圾回收机制。相辅相成的通过学习Python内部的垃圾回收机制，并了解其原理，可以使得开发者能够更好的写代码。

垃圾回收算法

标准CPython实现的垃圾收集器有两部分组件构成，一部分为引用计数模块(reference counting collector), 另一部分为分代垃圾回收器(generational garbage collector)。

引用计数算法(reference counting)

引用计数算法是最简单的垃圾回收算法：即对象当没有引用指向时，会被回收(deallocated)。
在Python的每一个对象中，都有一个 Py_ssize_t ob_refcnt 变量，用于标识当前对象被引用的次数：

引用计数器导致引用计数 +1 的情况：

  • 创建对象
  • 赋值操作
  • 函数传参
  • 对象作为元素被容器引用

导致引用计数 -1 的情况：

  • 引用对象的变量被显式销毁，例如del a
  • 引用对象的变量被赋予新的对象，例如a=24
  • 一个对象离开它的作用域（函数执行完毕时，函数中的局部变量）
  • 对象所在的容器被销毁，或从容器中删除对象

其次，还有**_PyObject_HEAD_EXTRA 变量，用于构造双向链表(refchain)**的，Python中每创建一个对象就会加入该链表中，当有垃圾变量被回收时就会移除该链表：


我们可以从下面的例子中，看一下对象的引用计数器增减情况，以及对象被销毁的过程：

import weakref
import sys

s1 = {1, 2, 3}
print(sys.getrefcount(s1)) # 2 -> 一个来自s1变量,一个来自getrefcount函数
s2 = s1
print(sys.getrefcount(s1)) # 3 -> 新增了一个s2变量

def bye():
    print("Gone with the wind....")

# weakref.finalize是弱引用不会增加引用计数器值
ender = weakref.finalize(s1, bye)  # 在s1的引用对象上,注册bye()函数

del s1  # --> del不删除对象,而是删除对象的引用
print(sys.getrefcount(s2)) # 2 --> 引用对象的计数器-1

s2 = "gg"  # 给s2重新绑定一个引用 --> {1, 2, 3}对象的计数器-1
# 此时{1, 2, 3}对象的引用计数器值为0  --> 调用回调函数bye():Gone with the wind....

标记清除(Mark and Sweep)

当存在下面的情况时(循环引用)，引用计数算法无法很好的完成垃圾回收工作：

import sys

v1 = [11, 22, 33]  # 列表对象1
v2 = [44, 55, 66]  # 列表对象2 

v1.append(v2) 
v2.append(v1)

del v1
del v2

# 此时已经没有变量指向对象列表1和列表2了,这两个对象应该被当作垃圾回收
# 但是因为上面的循环引用的操作,导致这两个对象的计数器值还为1,导致通过gc算法无法回收这两个对象

标记清除算法是一种基于追踪回收(tracing GC)技术实现的垃圾回收算法。主要由两段操作构造：标记(Mark)和清除(Sweep)。
标记清除算法作为 Python 的辅助垃圾收集技术主要处理的是一些容器对象，比如 list、dict、tuple，instance 等可能造成循环引用问题的对象，Python 使用一个双向链表将这些容器对象组织起来。定期会对这些对象进行扫描，如果存在循环引用的就使双方的引用计数器-1。
标记清除标记清除（Mark—Sweep）』算法是一种基于追踪回收（tracing GC）技术实现的垃圾回收算法。它分为两个阶段：第一阶段是标记阶段，GC 会把所有的『活动对象』打上标记，第二阶段是把那些没有标记的对象『非活动对象』进行回收。

对象之间通过引用（指针）连在一起，构成一个有向图，对象构成这个有向图的节点，而引用关系构成这个有向图的边。从根对象（root object）出发，沿着有向边遍历对象，可达的（reachable）对象标记为活动对象，不可达的对象就是要被清除的非活动对象。根对象就是全局变量、调用栈、寄存器。 mark-sweepg 在上图中，我们把小黑圈视为全局变量，也就是把它作为 root object，从小黑圈出发，对象 1 可直达，那么它将被标记，对象 2、3 可间接到达也会被标记，而 4 和 5 不可达，那么 1、2、3 就是活动对象，4 和 5 是非活动对象会被 GC 回收。

标记清除算法的优点在于可以解决循环引用的问题，并且在整个算法执行的过程中没有额外的开销。
标记清除算法的最主要缺点在于当执行标记清除时正常的程序将会被阻塞，另外一个缺点在于标记清除算法在执行很多次数后，会在程序的堆空间会产生一些小的内存碎片：

分代回收(Generational garbage collector）

分代回收算法是基于标记清除的，主要解决循环引用或自引用的问题。
首先GC分类器会将可能存在循环引用的对象分为三个双向链表（三代），每个新创建的对象属于第0代，每当对象在某代的垃圾回收中存活下来，就移至下一代。较低的代会更加频繁的进行垃圾回收（大部分对象在创建后很快就会消亡）。
在标准Python 3.6版本中通过gc.get_threashold函数可以获得每代的阈值：700、10、10。当代中对象数量达到阈值后就会进行一次垃圾回收。

总结

Python的垃圾回收以引用计数算法为主，标记清除和分代回收算法为辅
