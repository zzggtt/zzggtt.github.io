前置知识-WSGI

Web请求流程图：

首先我们需要思考一个问题，为什么 Flask/Django 编写的Web应用程序在 uWSGI 或者 gunicorn 上都可以运行呢？
答案就是它们都遵循了一个标准的接口，即 WSGI-PEPE3333 ，基于WSGI协议，可以促进Web服务器和Web应用程序或框架之间的可移植性：

This document specifies a proposed standard interface between web servers and Python web applications or frameworks, to promote web application portability across a variety of web servers.

下面是基于WSGI实现的简单的例子：

from wsgiref.simple_server import make_server

HELLO_WORLD = b"Hello world!\n"

"""
application：应用程序或者框架，是一个可调用的对象
接受两个参数：
   -- environ: 包含WSGI环境信息的字典
   -- start_response: 生成响应的WSGI回调函数,接受两个参数status和response_headers
"""
def application(environ, start_response):
    status = '200 OK'
    response_headers = [('Content-type', 'text/plain')]
    start_response(status, response_headers)
    return [HELLO_WORLD]


if __name__ == "__main__":
    # 基于wsgiref.simple_server模块实现一个简单的Web Server
    # 客户端的请求转发到application进行处理
    httpd = make_server('127.0.0.1', 9999, application)
    print("Serving HTTP on 127.0.0.1 port 9999...")
    httpd.serve_forever()
